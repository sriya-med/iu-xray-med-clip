# Reducing False Negatives in Medical Contrastive Learning

Fine-tuning CLIP on chest X-rays with clinically-aware InfoNCE loss functions to reduce false negatives in radiology vision-language models.

---

## Overview

This project investigates whether modifying the contrastive loss function to account for semantic similarity between radiology reports improves image-text retrieval on the IU X-Ray dataset. I implemented and compare three training objectives:

- **Baseline**: Standard InfoNCE (fine-tuned CLIP)
- **Masked InfoNCE**: Suppresses semantically similar pairs from the softmax denominator using MeSH label Jaccard similarity
- **Soft Target InfoNCE**: Reweights negatives using MeSH similarity as a soft label distribution

I evaluated using both standard Recall@K (exact-match) and a proposed **Semantic Recall@K** metric that accounts for clinical label overlap.

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/sriya-med/iu-xray-med-clip.git
cd medclip-iu
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate        # Mac/Linux
# .venv\Scripts\activate         # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download the IU X-Ray dataset

Go to [https://openi.nlm.nih.gov/faq?download=true](https://openi.nlm.nih.gov/faq?download=true) and download:

- **PNG images**: `NLMCXR_png.tgz`
- **Reports**: `NLMCXR_reports.tgz`

Extract both archives:

```bash
tar -xzf NLMCXR_png.tgz
tar -xzf NLMCXR_reports.tgz
```

Move the extracted folders into the `data/` directory:

```bash
mv NLMCXR_png medclip-iu/data/images
mv ecgen-radiology medclip-iu/data/reports
```

You should now have:
- `data/images/` — ~7,470 PNG files
- `data/reports/` — ~3,955 XML files

### 5. Parse the dataset

```bash
python src/parse_data.py
```

This generates `data/dataset.csv` with 3,826 rows containing image paths, impression text, findings text, and MeSH labels. Confirm the output looks like:

```
Total samples: 3826
```

## Training

All training commands should be run from the project root.

### Baseline (Standard InfoNCE)

```bash
python src/train.py --loss baseline --epochs 5 --batch_size 32
```

### Masked InfoNCE

```bash
python src/train.py --loss masked --epochs 5 --batch_size 32
```

### Soft Target InfoNCE

```bash
python src/train.py --loss soft --epochs 5 --batch_size 32
```

Checkpoints are saved to `results/{loss}_bs{batch_size}_best.pt` based on best validation loss.

### Key arguments

| Argument | Default | Description |
|---|---|---|
| `--loss` | `baseline` | Loss function: `baseline`, `masked`, or `soft` |
| `--epochs` | `5` | Number of training epochs |
| `--batch_size` | `32` | Batch size (use 64+ on GPU for stronger results) |
| `--lr` | `1e-5` | Learning rate |
| `--threshold` | `0.5` | Jaccard threshold for masked loss |
| `--tau` | `0.5` | Temperature for soft target loss |

### Running on GPU (recommended for batch_size=64)

On Google Colab or any CUDA machine, the device is detected automatically. To run all three variants at batch size 64:

```bash
python src/train.py --loss baseline --epochs 5 --batch_size 64 && \
python src/train.py --loss masked   --epochs 5 --batch_size 64 && \
python src/train.py --loss soft     --epochs 5 --batch_size 64
```

## Evaluation

```bash
python src/eval.py --checkpoint results/baseline_bs32_best.pt \
                   --out results/baseline_bs32_metrics.json
```

This reports both standard Recall@K (exact-match) and Semantic Recall@K (MeSH label overlap) for image-to-text and text-to-image retrieval.

## Results

| Loss | Batch Size | Std R@1 | Sem R@1 | Sem R@5 |
|---|---|---|---|---|
| Baseline | 32 | 1.04% | 21.67% | 42.56% |
| Masked   | 32 | 0.78% | **33.16%** | 41.51% |
| Soft     | 32 | 0.78% | 32.38% | **43.86%** |
| Baseline | 64 | 1.04% | 19.32% | 42.82% |
| Masked   | 64 | 0.26% | 22.72% | 40.73% |
| Soft     | 64 | 0.26% | **36.03%** | 41.78% |

Key finding: Standard Recall@K penalizes the model for retrieving semantically equivalent reports. Semantic Recall@K reveals that Masked InfoNCE achieves a **53% relative improvement** over baseline at batch size 32, and Soft Target InfoNCE achieves **87% relative improvement** at batch size 64.
